【ファイル添付ボタン（📎 ファイルを添付）】

- 対応形式：.txt, .pdf, .ppt, .pptx, .xlsx, .xls, .mp3, .wav, .mp4, .mov

【送信ボタン】
- クリック → 入力テキストをAIに送信
- AIの返答を待機中は「考え中...」と表示
→ 処理ID: API-CHAT-002（Claude APIでロープレ構築チャット）

【ロープレ生成ボタン（🎭 ロープレ生成）】
- クリック → ファイル選択ダイアログを表示


【ファイルアップロード後の自動処理フロー】
1. チャットエリアにファイル名とサイズ表示（ユーザーメッセージとして）
2. AI質問「どのタイプのデータですか？」と5つの選択ボタン表示：
   - 📖 見本データ（商談や接客の正解例）
   - 📚 教材データ（学ばせたい内容の資料）
   - 🏢 自社データ（商品情報や会社概要）
   - 👥 顧客データ（想定顧客やペルソナ）
   - 📄 その他
3. ボタン選択 → ユーザーの選択を表示
4. AI確認「承知しました。「ファイル名」を〇〇データとして処理します。ファイルを解析中です...」
5. ファイルタイプに応じた処理を開始

【ファイル処理の詳細（ファイルタイプ別）】

テキストファイル（.txt）：→ 処理ID: FILE-001
- FileReaderでテキストを直接読み込み
- UTF-8エンコーディング

PDFファイル（.pdf）：→ 処理ID: FILE-002
- PDF.jsライブラリを遅延ロード（初回のみ）
- 各ページを画像化（scale=2.0の高解像度）
- GPT-4 Visionで画像を解析し、テキスト・表・図を抽出
- ページごとに進捗表示「ページ X/Y を解析中...」
- 全ページ処理後、結合して返却

PowerPointファイル（.ppt, .pptx）：→ 処理ID: FILE-003
- JSZipライブラリを遅延ロード（初回のみ）
- XMLからテキスト要素（a:t）を抽出
- スライドごとに「=== スライド X ===」で区切り
- 処理後に警告表示「⚠️ 【推奨】完全な解析のため、PowerPointをPDFに変換してアップロードしてください」

Excelファイル（.xlsx, .xls）：→ 処理ID: FILE-004
- XLSXライブラリを遅延ロード（初回のみ）
- 各シートをCSV形式で抽出
- シートごとに「=== シート X: シート名 ===」で区切り

音声ファイル（.mp3, .wav）：→ 処理ID: FILE-005
- バックエンドサーバー経由でWhisper APIに送信
- 文字起こし結果をテキストとして返却

動画ファイル（.mp4, .mov）：→ 処理ID: FILE-006
- FFmpegライブラリを遅延ロード（初回のみ）
- 進捗表示「動画から音声を抽出中...」
- 動画から音声をMP3形式で抽出
- 抽出した音声をWhisper APIで文字起こし
- 進捗表示「音声を文字起こし中...」

【ファイル処理完了後】
- チャットに「ファイルの解析が完了しました！「ファイル」タブで内容を確認できます。」と表示
- ファイルタブにテキスト化されたデータを追加

プレイコンポーネント ＞ モード選択
................................................................................

【モードボタン（4つ、縦並び）】
- 台本モード
- お手本モード
- 確認モード（初期状態でアクティブ）
- 実戦モード

【各モードの役割】
- 台本モード：AIがお手本の会話を実演。ユーザーは聞くだけで学習。
- お手本モード：AIが顧客役、ユーザーに最適な発言例を提示しながらサポート。
- 確認モード：AIが顧客役、ユーザーが自由に話す。会話後にフィードバック。
- 実戦モード：AIが本物の顧客として振る舞い、難しい質問や反論も含む。

【モード選択時の挙動】
- ボタンクリック → 選択したモードをアクティブ表示
- 他のボタンは非アクティブ表示に戻る

プレイコンポーネント ＞ 再生エリア
................................................................................

【キャラクター表示ウィンドウ】
- 動画未設定時：絵文字アイコン表示（👔など）
- 動画設定時：動画を表示
- 状態に応じて動画切り替え（IDLE/SPEAKING）

【接続ステータス表示】
- 未接続時：赤色インジケーター + 「未接続」
- 接続中：黄色インジケーター + 状態メッセージ
  - 「エフェメラルキー取得中...」
  - 「WebSocket接続中...」
  - 「セッション作成待機中...」
  - 「マイク初期化中...」
- 接続完了時：緑色インジケーター + 「会話可能」
- 会話中：緑色インジケーター + 状態メッセージ
  - 「聞いています...」
  - 「AI応答中...」

【スタートボタン（▶ スタート / ■ 停止）】
- 未開始時：「▶ スタート」表示
- クリック（未開始時）→ ロープレセッション開始 → 処理ID: REALTIME-001
  1. エフェメラルキー取得（/api/realtime-session）→ 処理ID: API-REALTIME-001
  2. WebSocket接続（OpenAI Realtime API）→ 処理ID: API-REALTIME-002
  3. セッション設定更新（音声モード、VAD設定）
  4. マイク初期化 → 処理ID: REALTIME-002
  5. 「会話可能」状態に移行
- 開始後：「■ 停止」表示、ボタンスタイル変更
- クリック（開始後）→ ロープレセッション停止 → 処理ID: REALTIME-005
  1. WebSocket切断
  2. AudioContext停止
  3. メディアストリーム停止
  4. 「未接続」状態に戻る

【マイクボタン（🎤 OFF / 🎤 録音中...）】
- 未開始時：非アクティブ状態
- セッション開始後：アクティブ状態
- VADモード使用時：自動で音声検出、手動操作不要
- 手動モード時：
  - クリック（OFF時）→ 録音開始、「録音中...」表示
  - クリック（録音中）→ 録音停止、AIに送信

【結果オーバーレイ（ロープレ終了時に表示）】
- スコア表示（例：85点）
- フィードバックボックス
  - 良かった点（✓マーク）
  - 改善点
- 5秒後に自動非表示

【3つのタブ】

【会話の流れタブ】
初期状態：空

ロープレ生成後：
- 各モードの台本をカード形式で表示
- 台本カード構成：

  - 編集ボタン
  - タイムスタンプ
  - シーン設定情報
  - プレイヤー/相手情報
  - 会話テーブル：
    - 各行：話者（アイコン+名前）、発言内容
    - 話者アイコン：プレイヤー（👤）、相手（💼）、ナレーター（📢）
  - 全文表示ボタン

台本カードの操作：
- カード全体クリック → 展開/折りたたみ
- テキストセルダブルクリック → インライン編集開始
- 編集中は話者セルが「💾 保存」表示に変更
- 保存クリック → 編集内容を確定

【ロープレ設計タブ】
- ロープレの構造を視覚的に表示
- （詳細は別途実装）

【設計書タブ】
初期状態：4つのモードカードを空状態で表示

ロープレ生成後：
- 各モードのプロンプトをカード形式で表示
- プロンプトカード構成：
  - ヘッダー：🎯 モード名プロンプト
  - 編集ボタン
  - タイムスタンプ
  - プロンプト内容（折りたたみ状態）
  - 全文表示ボタン

プロンプトカードの操作：
- カード全体クリック → 展開/折りたたみ
- コンテンツエリアダブルクリック → 編集モード開始
- 編集ボタンクリック → 編集モードトグル
  - 編集中：contenteditable=true、ボタン「保存」表示
  - 保存時：contenteditable=false、ボタン「編集」表示、グローバル変数に保存


【ファイル選択ダイアログ】（🎭 ロープレ生成ボタンクリック時に表示）
- タイトル：「参照するファイルを選択（任意）」
- ファイルリスト：
  - アップロード済みファイルをチェックボックス付きで表示
  - 各行：チェックボックス、ファイル名、ファイルタイプタグ
- ファイルがない場合：「ファイルはアップロードされていません。チャット内容のみから生成します。」
- ダイアログ操作：
  - ファイル行クリック → チェックボックストグル
  - 生成ボタン → ロープレ生成処理開始
  - キャンセル/ダイアログ外クリック → ダイアログ閉じる

【ロープレ生成処理フロー】→ 処理ID: AGENT-ORCH-001
1. ダイアログから選択ファイルを取得
2. チャット履歴を収集
3. ダイアログを閉じる
4. 処理中メッセージを表示（スピナー付き）
5. エージェントオーケストレーターに送信 → 処理ID: API-AGENT-001
6. 進捗更新「ロープレ設計を分析中...」「台本を生成中...」
7. 3つの台本を並列生成 → 処理ID: AGENT-002, AGENT-003, AGENT-004
   - 台本モード、ポイントモード、実戦モード
8. 4つのシステムプロンプトを並列生成 → 処理ID: AGENT-005, 006, 007, 008
   - 台本/お手本/確認/実戦モード
9. 台本を「会話の流れ」タブに表示
10. プロンプトを「設計書」タブに表示
11. 処理中メッセージを削除
12. 完了メッセージをチャットに追加

サマリー画面 ＞ フィルターパネル（左側）→ 処理ID: CSV-003
--------------------------------------------------------------------------------

【レッスン絞り込み】
- 「レッスンを選択」ボタン
- クリック → ドロップダウン表示
- ドロップダウン構造（階層式）：
  - カテゴリー（チェックボックス、展開/折りたたみ可能）
    - レベル（チェックボックス、展開/折りたたみ可能）
      - レッスン（チェックボックス）
- チェック状態変更 → 即座にフィルタリング適用
- 親要素チェック → 子要素も全てチェック
- 子要素一部チェック → 親要素は中間状態

【プレイヤー絞り込み】
- 「プレイヤーを選択」ボタン
- クリック → ドロップダウン表示
- ドロップダウン構造（階層式）：
  - Org（チェックボックス、展開/折りたたみ可能）
    - Account（チェックボックス、展開/折りたたみ可能）
      - Group（チェックボックス、展開/折りたたみ可能）
        - Player（チェックボックス）
- チェック状態変更 → 即座にフィルタリング適用

【期間絞り込み】
- 開始月：月選択入力（input type="month"）
- 「〜」テキスト
- 終了月：月選択入力（input type="month"）

【期間の初期値と挙動の詳細】
- データ読み込み時に自動設定
- 初期値計算ロジック：
  1. 全データから最新日付を取得
  2. 終了月 = 最新日付の属する月
  3. 開始月 = 最新日付の6ヶ月前の月
- 例：最新日付が2025-01-15の場合
  - 終了月 = 2025-01
  - 開始月 = 2024-07

【期間表示範囲の詳細ルール】
- 選択された範囲にデータが存在する場合：その範囲のデータを表示
- 選択された範囲が6ヶ月未満の場合：
  - データ範囲が6ヶ月未満でも、グラフは最低6ヶ月分の横軸を表示
  - 不足分は開始月より前に空の期間として追加
- 例：データが2025-01のみの場合
  - 選択範囲：2025-01〜2025-01
  - グラフ表示：2024-08〜2025-01（6ヶ月分、2024-08〜2024-12は空データ）

================================================================================
付録C：OpenAI Realtime API連携詳細
================================================================================

【セッション開始フロー】
1. /api/realtime-sessionにPOSTリクエスト
2. エフェメラルキー（client_secret）を取得
3. WebSocket接続を開始（wss://api.openai.com/v1/realtime）
4. 認証プロトコル設定（openai-insecure-api-key.{key}）
5. session.createdイベント受信
6. session.updateメッセージ送信
   - modalities: ['audio', 'text']
   - voice: 選択された音声タイプ
   - input_audio_format: pcm16
   - output_audio_format: pcm16
   - turn_detection: server_vad設定
7. session.updatedイベント受信
8. マイク初期化
9. 「会話可能」状態

【音声送信処理】
1. ScriptProcessorNodeで音声データ取得
2. Float32をPCM16に変換
3. Base64エンコード
4. input_audio_buffer.appendメッセージで送信

【音声受信処理】
1. response.audio.deltaイベント受信
2. Base64デコード
3. PCM16をFloat32に変換
4. AudioBufferを作成
5. AudioBufferSourceで再生
